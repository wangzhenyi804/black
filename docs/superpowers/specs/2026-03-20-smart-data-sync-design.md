# 智能数据同步工作台设计文档 (Smart Data Sync Workspace Spec)

## 1. 业务目标
管理员每天需要录入来自不同渠道（微信截图、手机相册、CSV、Excel）的代码位统计数据。目前的流程效率低且归属人设置繁琐。本方案旨在建立一个**统一的智能数据同步工作台**，支持图片与文件的统一预览、编辑、自动补全（新建媒体/代码位）以及批量设置归属人，实现数据录入的全自动化与精准化。

## 2. 核心架构与数据流

### 2.1 整体架构
- **前端 (React)**: 统一导入入口，支持多种文件格式预览。提供全局配置区（如设置归属人）和明细编辑区。
- **后端 (Spring Boot)**: 
  - 集成 OCR 解析图片数据。
  - 统一解析引擎处理 CSV/Excel/图片识别结果。
  - 自动创建逻辑：若代码位或媒体不存在，则按预设规则自动新建。
- **持久层 (MySQL)**: 维护映射记忆表及核心业务表。

### 2.2 数据处理流程 (Pipeline)
1. **输入层**: 图片 (微信截图/相册)、CSV 或 Excel 文件。
2. **预览解析层 (后端)**:
   - **文件/图片统一解析**: 提取原始数据行。
   - **启发式分类与校准**: (同前述逻辑，自动识别指标、反算 ecpm)。
   - **名称匹配与智能记忆**: 优先匹配历史纠正名称。
   - **自动识别缺失项**: 标记系统中不存在的代码位或媒体（待新建状态）。
3. **展示与干预层 (前端)**: 
   - 识别结果进入统一预览表。
   - **批量归属人设置**: 管理员可在预览页顶部选择一个“归属人”，该设置将应用于本批次所有新建的代码位和导入的数据。
   - **手动微调**: 修正数据、补齐日期、确认/更改自动创建的媒体信息。
4. **持久化层**: 
   - 按需新建媒体 -> 新建代码位。
   - 批量写入 `stats` 表。
   - 更新 `ocr_mapping_history` 映射表。

### 2.3 核心数据表设计
#### `ocr_mapping_history` (映射记忆表)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BigInt | 主键 |
| `original_text` | String | OCR 识别到的原始文本（如“新 1”） |
| `target_slot_name` | String | 关联的系统逻辑代码位名称 |
| `last_used_at` | DateTime | 最后一次使用时间 |

## 3. 功能详述

### 3.1 统一同步工作台 (frontend)
- **多格式支持**: 拖拽区同时接收 `.jpg`, `.png`, `.csv`, `.xlsx`。
- **全局配置面板**:
  - **日期设置**: 若识别结果缺失日期，支持一键填充。
  - **归属人选择**: 提供用户下拉列表，设置该批次数据的“归属人”。
- **智能预览表格**: 
  - 状态标识：[已匹配 / 待新建媒体 / 待新建代码位]。
  - 支持内联编辑。
  - 数据追加：支持在预览过程中继续添加文件/图片。

### 3.2 后端解析逻辑 (backend)
- **`POST /stats/preview`**: 支持 `MultipartFile`（图片或表格），统一返回 `List<PreviewRowDTO>`。
- **自动补全逻辑**:
  - 若 `target_slot_name` 在系统中不存在，预览结果标记 `isNewSlot = true`。
  - 自动解析媒体名称（或域名），若不存在标记 `isNewMedia = true`。
- **`POST /stats/confirm-import`**: 接收前端确认后的最终 JSON 数据（包含归属人 ID）。
  - 事务化处理：新建缺失实体 -> 写入统计数据 -> 更新映射记忆。

### 3.3 启发式规则详解 (修订版)
| 字段 | 规则优先级 | 容错处理 |
| :--- | :--- | :--- |
| **展现量** | `Max(Integers)` | 若有多个接近，标黄提示人工检查 |
| **点击量** | `SecondMax(Integers)` | 同上 |
| **收入** | `Max(Decimals)` (且非日期/非点击率) | 结合位置信息，优先取展现量右侧的小数 |
| **点击率** | 匹配 `[0-9.]+%` | |
| **代码位** | 优先寻找位置最靠左/上的非标字符串 | 提供下拉搜索框供人工修正 |

## 4. 技术选型
- **OCR 服务**: 阿里云通用文字识别/表格识别 API。
- **文件解析**: Apache POI (Excel), OpenCSV (CSV)。
- **匹配算法**: Levenshtein Distance。
- **前端组件**: `react-dropzone`, `lucide-react`, `CustomEditableTable`。

## 5. 验收标准 (Checklist)
- [ ] 统一工作台支持图片、CSV、Excel 同时拖入识别。
- [ ] 预览界面提供“批量设置归属人”下拉框，并成功同步到入库数据。
- [ ] 系统能自动识别并创建不存在的代码位和媒体（继承 CSV 导入的自动创建逻辑）。
- [ ] 具备代码位名称映射记忆功能。
- [ ] 预览表格支持多行追加、删除及所有字段的编辑。
- [ ] 自动根据收入/展现/点击校准 ecpm 和 acp。

## 6. 风险与规避
- **归属人误设**: 在“确认导入”前增加二次确认弹窗，展示所选归属人。
- **重名冲突**: 自动创建实体前先进行严格的名称查重。
- **识别费用**: 前端图片压缩处理。
