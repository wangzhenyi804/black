# 自定义下拉框样式优化计划

## 问题分析
当前“代码位管理”页面中的下拉框（例如筛选媒体、筛选类型）使用了浏览器默认的原生 `<select>` 样式，导致在不同操作系统下（尤其是 macOS）会出现白底或不协调的 UI，与项目整体的深色（Zinc/Black）风格不符。虽然已经尝试通过 CSS 类名修改，但原生 `<select>` 在某些场景下的可定制性仍然有限。

## 实施步骤

### 1. 创建自定义 Select 组件
- [ ] 在 `src/components` 目录下新建 `Select.tsx` 组件。
- [ ] 使用 Headless UI（或手动实现）构建完全自定义的下拉菜单。
- [ ] **样式规范**：
    - 触发器（Button）：背景 `bg-zinc-900/50`，边框 `border-white/5`，文字 `text-zinc-300`。
    - 下拉面板（Options）：背景 `bg-zinc-900`，毛玻璃效果 `backdrop-blur`，阴影 `shadow-xl`。
    - 选项项（Option）：悬停高亮 `hover:bg-primary/10 hover:text-primary`，选中状态高亮。
    - 交互：支持点击展开/收起，点击外部关闭。

### 2. 替换 CodeSlots.tsx 中的原生 Select
- [ ] 替换“所有媒体”筛选下拉框。
- [ ] 替换“所有形式”筛选下拉框。
- [ ] 替换新增弹窗中的“所属媒体”选择框。

### 3. 替换 Media.tsx 中的原生 Select
- [ ] 替换“行业类别”筛选下拉框。
- [ ] 替换新增弹窗中的“行业类别”选择框。

### 4. 替换 Users.tsx 中的原生 Select
- [ ] 替换创建/编辑用户时的“角色选择”下拉框。

### 5. 样式与功能验证
- [ ] 确保下拉框在暗色模式下显示正常，无白色背景闪烁。
- [ ] 验证选项点击后能正确回填值并触发 `onChange` 事件。
- [ ] 验证 z-index 层级，确保下拉内容浮动在表格上方。
