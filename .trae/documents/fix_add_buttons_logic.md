# 修复新增媒体与新增代码位按钮无响应的计划

## 问题分析
在之前的 UI 重构过程中，虽然保留了侧边栏和基础布局，但在重写 `Media.tsx` 和 `CodeSlots.tsx` 时，遗漏了用于新增记录的模态框（Modal）及其相关的提交逻辑。导致点击“新增”按钮时，状态虽然改变但没有对应的 UI 组件弹出。

## 实施步骤

### 1. 修复媒体管理页面 (Media.tsx)
- [ ] 补全 `handleSubmit` 逻辑，用于提交新增媒体的表单数据。
- [ ] 在页面末尾添加现代化的暗色风格模态框 JSX。
- [ ] 确保表单项（媒体名称、域名、分类、ICP 备案、备注）与后端字段匹配。

### 2. 修复代码位管理页面 (CodeSlots.tsx)
- [ ] 为 `新增代码位` 按钮添加 `onClick={() => setIsModalOpen(true)}` 事件。
- [ ] 定义 `formData` 状态及其对应的 `Partial<CodeSlot>` 类型。
- [ ] 实现 `handleSubmit` 逻辑，调用 `/codeslots/` 接口。
- [ ] 在页面末尾添加对应的表单模态框，包含所属媒体选择、展现形式、终端类型等字段。

### 3. 样式同步
- [ ] 确保模态框采用 `Zinc` 暗色主题，搭配透明毛玻璃背景（backdrop-blur）。
- [ ] 统一按钮、输入框、下拉框的样式，符合全站现代化重构风格。

### 4. 功能验证
- [ ] 在预览界面点击“新增媒体”，确认模态框弹出并能正常输入。
- [ ] 在预览界面点击“新增代码位”，确认模态框弹出并能正常选择所属媒体。
