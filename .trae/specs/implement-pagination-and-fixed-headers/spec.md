# 列表分页与固定表头功能规范

## 为什么
当前系统的列表页面（如媒体管理、代码位管理、数据概览等）在滚动时，顶部的搜索/筛选区域以及表格的表头会随之消失，导致用户在查看长列表时难以进行筛选操作或辨认列含义。此外，需要统一和完善分页功能，提升用户体验。

## 修改内容
- **后端支持**：确保所有列表接口均支持 `page` 和 `size` 参数，并返回分页结果（已基本实现，需检查遗漏）。
- **前端布局调整**：
    - **区域固定**：搜索/筛选区域、操作按钮区域（如“创建网站”）在页面滚动时保持固定。
    - **表头固定**：表格的 `thead` 在滚动时保持固定在容器顶部。
    - **局部滚动**：将滚动条限制在数据表格区域，而不是整个页面。
- **分页组件统一**：使用标准的、包含页码跳转和每页条数切换的分页组件。

## 影响范围
- **受影响页面**：
    - 媒体管理 (`Media.tsx`)
    - 代码位管理 (`CodeSlots.tsx`)
    - 用户管理 (`Users.tsx`)
    - 数据概览 (`DataOverview.tsx`)
    - 代码位数据 (`CodeSlotData.tsx`)
- **受影响组件**：
    - `Pagination.tsx` (需优化)
    - `Layout.tsx` (布局容器)

## 新增需求
### 需求：固定头部与表头
系统应提供一种布局，使得页面顶部的筛选条件和表格头在数据滚动时始终可见。

#### 场景：成功案例
- **当** 用户进入媒体管理页面并向下滚动列表时
- **那么** 顶部的“网站名称”、“分类”筛选框以及表格的“网站ID”、“网站名称”等表头应保持在原位不移动。

## 修改需求
### 需求：标准分页功能
分页组件应包含：当前页码、总条数、每页条数切换（10, 20, 30, 50）、快速跳转至特定页。
