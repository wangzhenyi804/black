# 代码位数据批量导入与系数管理开发计划

本计划旨在为“代码位数据”模块增加批量导入功能，并引入管理员可控的“分成系数”管理机制。

## 1. 核心变更点

### 后端变更 (Architect-001)
- **数据库架构更新**：在 `stats` 表中新增 `pre_revenue`（分成前收入）、`ratio`（系数）和 `extra_data`（额外原始数据，JSON格式）字段。
- **实体类更新**：修改 `Stats.java` 以匹配数据库变更。
- **导入接口开发**：在 `StatsController` 中实现 `POST /stats/import` 接口。
    - 仅限管理员访问。
    - 解析 CSV/Excel 数据：Excel 中的“收入”映射为 `pre_revenue`。
    - 自动计算：`revenue` (分成后收入) = `pre_revenue` * `ratio`。
    - 存储冗余数据：将 Excel 中非核心字段序列化为 JSON 存入 `extra_data`。
- **系数调整接口**：提供更新单个统计记录系数的接口，并同步重算分成后收入。

### 前端变更 (FE-001)
- **权限控制**：基于 `AuthContext` 识别管理员身份，仅对管理员显示导入按钮和系数编辑功能。
- **数据管理增强**：
    - 在 `CodeSlotData.tsx` 页面顶部增加“批量导入数据”按钮。
    - 实现导入文件的上传交互与进度反馈。
    - 将“系数”列改为可编辑状态（仅限管理员），支持即时调整。
- **列表显示优化**：确保表格正确渲染后端返回的 `preRevenue`、`ratio` 和 `revenue`。

## 2. 实施步骤

### 第一阶段：后端基础设施与 API
1.  **数据库变更**：执行 SQL 脚本，为 `stats` 表添加新字段。
2.  **实体类与 DTO 同步**：更新 `Stats` 实体和 `StatsCodeSlotDTO`。
3.  **实现导入逻辑**：编写 CSV 解析与数据转换逻辑，包含收入计算公式。

### 第二阶段：前端交互开发
1.  **新增导入按钮**：在代码位数据页面工具栏集成导入触发器。
2.  **开发系数编辑功能**：为管理员提供系数调整的 UI 入口。
3.  **联调测试**：验证文件上传、数据解析、收入计算及列表刷新流程。

### 第三阶段：质量保障 (QA-001)
1.  **数据准确性校验**：验证 `分成前收入 * 系数 = 分成后收入` 的逻辑。
2.  **安全性验证**：确保非管理员用户无法通过前端按钮或后端接口进行数据导入或系数修改。
3.  **容错性测试**：测试上传格式错误的 CSV 文件时的系统反馈。

## 3. 验收标准
- [ ] 管理员可通过 CSV 批量导入代码位统计数据。
- [ ] 导入时，系统能根据设定的系数（默认为 1.0 或由管理员指定）自动计算分成后收入。
- [ ] 原始 Excel 中的非展示字段已加密或以 JSON 形式安全存储在后端。
- [ ] 系数调整功能生效，且能即时更新关联的收入数据。
- [ ] 非管理员用户界面保持简洁，无导入及编辑权限。
