# 媒体管理功能实现计划

根据用户需求和提供的设计图，需要对现有的媒体管理功能进行全面升级。主要包括后端数据库结构的调整、API的增强（分页、筛选），以及前端页面的重构（筛选列表、复杂表单弹窗）。

## 1. 后端开发 (Backend)

### 1.1 数据库与实体更新
- **修改 `media` 表结构**:
  - 新增字段:
    - `domain` (网站域名)
    - `category` (行业类别)
    - `icp_code` (备案信息)
    - `daily_visits` (访问量，存储枚举值或字符串)
    - `description` (网站描述 - 已存在，需确认长度)
    - `stats_auth_type` (统计查看授权方式)
    - `agent_auth_url` (代理授权书图片路径)
    - `copyright_url` (软著图片路径)
    - `status` (状态: 待初审、初审未通过、待验证、验证失败、待复核、复核未通过、审核通过)
    - `rejection_reason` (审核未通过原因)
    - `note` (备注)
- **更新 `Media.java` 实体类**: 映射上述数据库字段。

### 1.2 接口开发
- **通用文件上传接口 (`UploadController`)**:
  - 实现 `/upload` 接口，用于前端上传“代理授权”和“软著”图片，返回文件访问URL。
- **媒体管理接口更新 (`MediaController`)**:
  - **分页与筛选列表 (`GET /media`)**:
    - 支持参数: `page` (页码), `size` (每页条数), `name` (模糊搜索), `category` (分类), `status` (状态)。
    - 返回结构: 包含总条数、当前页数据列表。
  - **创建/修改媒体 (`POST /media`, `PUT /media`)**:
    - 接收前端表单的所有字段。
    - 简单的状态流转逻辑（创建时默认为“待初审”）。

## 2. 前端开发 (Frontend)

### 2.1 类型定义
- 更新 `Media` 接口定义，包含所有新字段。
- 定义筛选条件接口 `MediaFilter`。

### 2.2 媒体列表页 (`Media.tsx`)
- **筛选区域**:
  - 网站名称输入框。
  - 分类下拉选择框。
  - 状态标签页 (Tabs): 全部、待初审、初审未通过、初审通过待验证、验证失败、验证通过待审核、审核未通过、审核通过。
  - 查询按钮。
- **数据表格**:
  - 展示列: 网站ID、网站名称、网站域名、分类、备案信息、状态 (带颜色样式)、审核未通过原因、创建时间、备注、操作。
  - 操作列: 修改、域名验证 (仅在特定状态显示)。
- **分页组件**:
  - 实现底部分页条。

### 2.3 创建/编辑网站弹窗
- **模态框组件**:
  - 封装一个可复用的 `Modal` 组件（如果项目中没有）。
- **表单实现**:
  - **网站域名**: 文本输入。
  - **网站名称**: 文本输入。
  - **行业类别**: 下拉选择。
  - **备案信息**: 文本输入。
  - **访问量**: 单选框 (1w以下, 1w-10w, ...)。
  - **网站描述**: 多行文本框。
  - **统计查看授权**: 单选框。
  - **代理授权/软著**: 图片上传组件 (调用后端的 `/upload` 接口)。
  - **备注**: 文本输入。
- **表单验证**: 必填项校验。
