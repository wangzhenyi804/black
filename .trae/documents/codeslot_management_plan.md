# 代码位管理模块设计文档

## 1. 需求分析与产品设计

### 1.1 背景与目标
代码位（Code Slot）是连接媒体与广告主的桥梁。媒体主在网站/应用中嵌入代码位，用于展示广告。当前系统仅实现了最基础的名称与代码存储，缺乏对广告形式、终端类型、尺寸、状态等核心业务属性的管理，无法满足实际运营需求。本模块旨在重构并增强代码位管理能力，提供全生命周期的管理功能。

### 1.2 核心功能需求
参考业界标准与用户需求，代码位管理需包含以下核心功能：

1.  **代码位列表与查询**
    *   **多维度筛选**：支持按 代码位ID、名称、所属媒体、终端类型（PC/Mobile/App）、展现类型（Banner/插屏等）、广告类型、状态（有效/暂停/审核中）进行筛选。
    *   **关键指标展示**：列表需展示 ID、名称、所属媒体、尺寸、反屏蔽状态、状态、更新时间等。
    *   **分页展示**：支持大量数据的分页浏览。

2.  **创建/编辑代码位**
    *   **基本信息**：所属媒体（关联 Media）、名称。
    *   **业务属性**：
        *   **终端类型**：PC Web、Mobile Web、App。
        *   **展现类型**：固定块（Banner）、信息流、插屏、全屏、原生等。
        *   **尺寸**：支持预设尺寸（如 300x250, 728x90）或自定义尺寸。
        *   **反屏蔽设置**：是否开启反屏蔽代码（开启后生成不同代码）。
    *   **代码生成**：系统根据配置自动生成 JS/SDK 代码片段，而非用户手动输入。
    *   **状态管理**：启用/停用开关。

3.  **代码获取**
    *   提供“获取代码”功能，一键复制生成的广告代码。

4.  **数据校验与权限**
    *   同一媒体下代码位名称不可重复。
    *   普通用户只能管理自己的代码位，管理员可管理所有。

## 2. 技术方案设计

### 2.1 后端技术方案 (Spring Boot)

#### 2.1.1 接口设计 (RESTful API)
*   `GET /codeslots/`
    *   参数：`page`, `size`, `name`, `mediaId`, `type`, `status`
    *   响应：分页列表数据
*   `GET /codeslots/{id}`
    *   响应：单条详情
*   `POST /codeslots/`
    *   请求体：`mediaId`, `name`, `type`, `terminal`, `width`, `height`, `isShielding`
    *   逻辑：校验 -> 生成唯一ID -> **调用代码生成服务** -> 落库
*   `PUT /codeslots/{id}`
    *   请求体：支持修改名称、状态、反屏蔽设置（尺寸/类型通常创建后不可改）
*   `DELETE /codeslots/{id}`
    *   逻辑：软删除或物理删除（需检查有关联数据）
*   `GET /codeslots/{id}/code`
    *   响应：生成的 JS 代码片段

#### 2.1.2 核心逻辑实现
*   **代码生成策略 (CodeGenerationService)**：
    *   实现一个简单的模板引擎，根据 `slotId` 和配置生成 `<script>` 标签或 SDK 初始化代码。
    *   示例：`<script src="https://ad.blackad.com/j?id={id}&w={w}&h={h}"></script>`

*   **数据权限**：
    *   复用已有的 `SecurityConfig` 和 `Controller` 层面的 `userId` 过滤逻辑。

### 2.2 前端技术方案 (React + Tailwind)

#### 2.2.1 页面结构 (`pages/CodeSlots.tsx`)
*   **筛选区 (Filter Bar)**：
    *   使用 Grid 布局，包含 输入框（ID/名称）、下拉框（终端/类型/状态）、查询按钮、重置按钮。
*   **操作区**：
    *   "新建代码位" 按钮，点击弹出模态框（或跳转新页面）。
*   **列表区 (Table)**：
    *   展示核心字段。
    *   操作列：获取代码（弹窗展示代码）、编辑、开关状态、删除。

#### 2.2.2 组件交互
*   **新建/编辑模态框**：
    *   **媒体选择**：异步加载当前用户的媒体列表。
    *   **联动逻辑**：选择“PC”时，展现类型选项变化；选择“Banner”时，尺寸输入框出现。
*   **代码展示弹窗**：
    *   只读文本域 + “一键复制”按钮。

## 3. 实施计划 (Implementation Plan)

### Phase 1: 后端重构
1.  **数据库迁移**：修改 `code_slot` 表，添加 `type`, `terminal`, `width`, `height`, `status`, `updated_at` 等字段。
2.  **实体更新**：更新 Java Entity 类。
3.  **Service层升级**：实现代码生成逻辑，完善 CRUD 校验。
4.  **Controller层升级**：支持复杂查询参数，对接新字段。

### Phase 2: 前端重构
1.  **UI 组件开发**：开发筛选栏、状态标签组件。
2.  **列表页升级**：对接新的查询接口，展示丰富信息。
3.  **表单页开发**：实现复杂的表单联动逻辑（创建/编辑）。
4.  **代码获取功能**：实现代码展示与复制。

### Phase 3: 联调与测试
1.  **权限测试**：确保普通用户只能操作自己的数据。
2.  **业务测试**：验证代码生成是否正确，状态切换是否生效。
