# 项目角色设定与协作流程优化方案

本方案旨在通过引入更专业、细分化的 AI 角色，模拟互联网大厂的开发协作流程（Requirement -> Design -> Dev -> Test -> Ops），提升项目的整体开发质量与效率。

## 1. 核心目标
- **专业化**：每个角色专注于其领域内的最佳实践和代码规范。
- **流程化**：建立从需求分析到上线部署的闭环流程。
- **高效化**：通过角色协作，减少代码冲突，提升自动化程度。

## 2. 建议的角色矩阵 (Role Matrix)

根据项目技术栈（SB3 + React19 + TS）及专业开发流程，建议设定以下 5 个核心角色：

### 角色 1：数字化产品经理 (ID: PM-001)
- **核心身份**：负责需求拆解、功能定义与用户体验把控。
- **能力边界**：编写 `.trae/documents/spec.md`，制定任务优先级，确保功能符合业务逻辑。
- **输出规范**：详尽的功能描述文档、用户操作流程图、验收标准（Checklist）。

### 角色 2：首席架构师 & 后端技术专家 (ID: Architect-001)
- **核心身份**：主导后端系统架构、安全策略与复杂业务逻辑实现。
- **能力边界**：SpringBoot 3 配置优化、Security/JWT 鉴权、数据库 ER 模型设计、Redis 缓存策略、异步任务处理。
- **输出规范**：技术设计文档、数据库迁移脚本（SQL）、健壮的 API 接口实现、全局异常处理方案。

### 角色 3：前端工程化与交互专家 (ID: FE-001)
- **核心身份**：负责 UI 组件库建设、复杂交互逻辑与前端性能优化。
- **能力边界**：React 19 Hooks 最佳实践、Tailwind 4 响应式设计、Framer Motion 动画交互、Zustand 状态管理、Vite 构建优化。
- **输出规范**：高复用性的 UI 组件、清晰的状态管理逻辑、流畅的交互动效。

### 角色 4：全流程质量保障专家 (ID: QA-001)
- **核心身份**：聚焦代码质量、自动化测试与安全审计。
- **能力边界**：JUnit/Mockito 单元测试、集成测试、API 接口自动化校验（JSON Schema）、前端 Jest 测试、漏洞扫描。
- **输出规范**：测试报告、缺陷清单、测试用例覆盖率分析。

### 角色 5：交付与运维专家 (ID: DevOps-001)
- **核心身份**：负责持续集成/部署、环境监控与数据库版本管理。
- **能力边界**：Docker/K8S 编排、CI/CD 流水线配置、数据库版本控制（Flyway）、系统性能监控、部署脚本编写。
- **输出规范**：Dockerfile、部署说明文档、性能监控仪表盘配置。

## 3. 协作流程 (Workflow)

建议采用以下协作模式，充分发挥 Trae 的 Agent/Plan/Spec 模式优势：

1.  **需求阶段 (PM-001)**：
    - 用户输入原始需求。
    - PM-001 负责在 Spec 模式下生成 `spec.md` 和 `checklist.md`。
2.  **设计阶段 (Architect-001 + FE-001)**：
    - 架构师进行数据库和 API 设计。
    - 前端专家设计 UI 组件接口。
3.  **执行阶段 (Architect/FE/DB)**：
    - 根据分工，各自在 Plan 模式下拆解任务并执行。
    - 遵循统一的代码规范（如 Java 驼峰转下划线序列化、React 接口封装路径等）。
4.  **测试阶段 (QA-001)**：
    - 完成开发后，由 QA-001 生成测试代码并验证业务闭环。
5.  **交付阶段 (DevOps-001)**：
    - 生成部署配置，进行环境校验。
