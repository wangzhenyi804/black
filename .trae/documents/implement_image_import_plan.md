# 图片导入代码位数据实施计划

该方案旨在通过集成阿里云 OCR 能力，支持用户通过拖拽图片的方式导入代码位统计数据，并提供前端预览、编辑及数据追加功能。

## 1. 当前现状分析

* **后端**: 已有 `/stats/import` 接口支持 CSV 导入，具备基础的代码位匹配和数据去重逻辑。

* **前端**: `CodeSlotData.tsx` 页面仅支持简单的 CSV 文件上传，缺乏预览和编辑环节。

* **痛点**: 无法直接处理图片格式的数据报表，手动输入效率低下且易出错。

## 2. 方案详述

### 后端实现 (Spring Boot)

1. **OCR SDK 集成**: 在 `pom.xml` 中引入阿里云 OCR (通用文字识别/表格识别) SDK。
2. **OCR 解析 Service (`OcrService`)**:

   * 调用阿里云 OCR 获取识别结果。

   * **逻辑增强**: 根据识别到的文字坐标进行水平分列。即使没有表头，也按照预设的顺序（展现、收入、点击、eCPM、CTR、CPC）提取数据。

   * **模糊匹配**: 提取图片中的红色字符串（代码位名称），利用模糊匹配算法（如 Levenshtein 距离）与数据库中的 `code_slot` 表进行比对。
3. **Controller 接口**:

   * `POST /stats/ocr-preview`: 接收图片文件，返回识别出的原始数据行（包含匹配到的代码位 ID、名称及各项指标）。

### 前端实现 (React + Tailwind)

1. **导入模态框升级**:

   * 增加“图片导入”选项。

   * 实现 **Drag & Drop** 拖拽上传区域，支持单次拖入单张或多张图片。
2. **预览确认表格**:

   * 实时显示 OCR 识别结果，以行展示。

   * **可编辑性**: 表格内所有单元格（日期、代码位名称、展现量等）均为可编辑状态。

   * **数据追加**: 支持用户再次拖入图片，新识别的数据自动追加在表格下方。
3. **数据提交**:

   * 用户核对无误后，点击“确认导入”，将预览表格中的最终数据批量发送至后端存入数据库。

## 3. 具体修改项

### 后端 (Java)

* **`pom.xml`**: 添加 `aliyun-java-sdk-core` 和 `aliyun-java-sdk-ocr` 依赖。

* **`com.blackad.backend.service.OcrService`**: 新建类处理 OCR 核心逻辑。

* **`com.blackad.backend.controller.StatsController`**:

  * 增加 `/ocr-preview` 接口。

  * 增加 `/batch-import` 接口（接收 JSON 数组形式的数据）。

### 前端 (TypeScript)

* **`frontend/src/components/ImportStatsModal.tsx`**: 新建组件，封装导入逻辑。

* **`frontend/src/pages/CodeSlotData.tsx`**: 替换原有的导入按钮，改为触发新的模态框。

## 4. 验证计划

1. **OCR 解析测试**: 使用多种不同布局（带/不带表头）的截图测试识别率。
2. **模糊匹配测试**: 测试当代码位名称有微小偏差时（如简写或拼写错误）的匹配成功率。
3. **全流程联调**: 从拖拽图片 -> 预览修改 -> 成功入库。

## 5. 待确认项

* 阿里云 OCR 具体使用的 API 版本（通用文字识别 vs 电商/表格识别）。

* 模糊匹配的相似度阈值设定（建议 0.8 以上）。
