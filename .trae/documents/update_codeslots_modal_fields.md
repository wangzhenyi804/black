# 修复“创建代码位”弹窗字段的计划

## 问题分析
当前“创建代码位”弹窗的字段较少，未能完全覆盖业务需求。根据用户提供的截图，需要增加展现形式、广告类型、广告形式、尺寸比例、样式配置、反屏蔽代码及图片上传等详细字段，并采用更直观的按钮组（Segmented Control）交互方式。

## 实施步骤

### 1. 更新数据模型与状态 (CodeSlots.tsx)
- [ ] 在 `CodeSlot` 接口中确认字段：`ad_type`, `display_type`, `style_type`, `is_shielding` 等。
- [ ] 定义常量选项映射：
    - `TERMINAL_TYPES`: H5, PC
    - `DISPLAY_FORMS`: 固定块, 底部悬浮, 插屏
    - `AD_TYPES`: 信息流, 图文广告, 搜索推荐
    - `AD_FORMS`: 原生缩略图, 原生图文, 原生三图, 原生大图, 原生文字链
    - `STYLE_CONFIGS`: 默认, 自定义
- [ ] 更新 `formData` 的初始状态。

### 2. 重构弹窗 UI (CodeSlots.tsx)
- [ ] **代码位名称**：保持现状。
- [ ] **终端类型**：改为按钮组（H5, PC）。
- [ ] **展现形式**：改为按钮组（固定块, 底部悬浮, 插屏）。
- [ ] **广告类型**：新增按钮组（信息流, 图文广告, 搜索推荐）。
- [ ] **广告形式**：改为按钮组（原生缩略图等 5 个选项）。
- [ ] **尺寸**：优化 `ratio` 输入框，添加 "20 :" 前缀及比例范围提示。
- [ ] **样式配置**：新增按钮组（默认, 自定义）。
- [ ] **反屏蔽代码**：新增单选框（使用/不使用），并附带业务场景描述。
- [ ] **图片上传**：新增图片上传按钮，复用已有的上传逻辑。
- [ ] **特殊备注**：保持现状。

### 3. 后端兼容性确认
- [ ] 检查 `CodeSlot.java` 实体类，确保字段名（如 `adType`, `displayType` 等）与前端 `formData` 匹配。
- [ ] 确认数据库 `schema.sql` 已包含这些字段。

### 4. 样式同步
- [ ] 确保按钮组在高亮状态下使用 `bg-primary/10 text-primary border-primary/20` 风格。
- [ ] 保持弹窗的暗色毛玻璃质感（backdrop-blur）。
